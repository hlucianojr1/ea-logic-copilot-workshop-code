## What this repo is

This repository is the **build system and source-of-truth** for a 9-session GitHub Copilot
workshop series Insight is delivering to EA. It is _not_ the demo workspace itself — it
**generates** the EA-facing demo workspace under [`output/ea-cpp-games/`](output/) plus all
facilitator and learner content under [`sessions/`](sessions/).

The work is driven by two inputs, both committed under [`docs/`](docs/):

- Bert Sandie's email outlining EA's four target topics (Bug Fixing, Architecture
  Understanding via EASTL, Spec-Driven Dev, Multi-Agent intro).
- The Mural priorities marked **Ready** (Spec-Kit Mastery, Agentic SDLC w/ HITL, Context
  Engineering at Scale, Coding/Cloud Agent workflows).

See [`docs/engagement-charter.md`](docs/engagement-charter.md) for full scope, audience, and
success metrics.

## How to build (humans)

Prerequisites: Node 20+, CMake 3.28+, a C++20 toolchain (MSVC on Windows, GCC 13+ or Clang 17+
on Linux/macOS), [GitHub CLI](https://cli.github.com), and VS Code with the GitHub Copilot
extensions enabled.

```bash
git clone git@github.com:Insight-NA/ea_copilot_workshop.git
cd ea_copilot_workshop
npm install
```

Open the folder in VS Code. The `.vscode/` settings load `.github/prompts/`, `.github/chatmodes/`,
and `.github/instructions/` automatically. From Copilot Chat:

1. Run `/engagement-charter` once to establish the canonical charter.
2. Run `/bootstrap-repo`, `/bootstrap-copilot-customization`, `/bootstrap-ci`, `/bootstrap-mcp`
   in order (only the first time, or after structural changes).
3. Run the Tier-1 (`factory-*`) and Tier-2 (`demo-workspace-*`) prompts in the order described
   in [`docs/build-graph.md`](docs/build-graph.md).
4. Run the Tier-3 session prompts (`/session-01-comprehension-at-scale`, etc.) — these are
   parallelizable.
5. Run the Tier-5 review prompts and resolve all Critical / Significant findings.

```bash
npm run lint            # markdownlint + prettier

```

CI runs all of the above on every PR. See [`.github/workflows/`](.github/workflows/).

## Running the demo-workspace build & tests

The C++ engine lives under [`output/ea-cpp-games/`](output/), and that is the **only** folder
that contains a `CMakePresets.json`. CMake and CTest read presets from the current working
directory, so every `cmake --preset` / `ctest --preset` command must run **from
`output/ea-cpp-games/`**. Running them from the repo root fails with:

```text
CMake Error: Could not read presets ... File not found: .../CMakePresets.json
```

Always `cd` into the workspace first:

```bash
cd output/ea-cpp-games
ctest --preset default-debug --output-on-failure
```

Optional convenience — add a personal alias to your own `~/.zshrc` (not committed) so the
command works from anywhere in the repo:

```bash
# ~/.zshrc — runs the engine_demo baseline tests from any directory
alias engine-test='(cd "$(git rev-parse --show-toplevel)/output/ea-cpp-games" && ctest --preset default-debug --output-on-failure)'
```
