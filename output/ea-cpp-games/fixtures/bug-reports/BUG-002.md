# BUG-002 — Game-loop accumulator drifts after extended runs

- **Severity:** medium (replay regression; does not crash)
- **Subsystem:** sim
- **File:** `src/engine_demo/sim/game_loop.cpp`
- **Class:** logic / determinism
- **Anchor for:** Session 03 (Logic Bugs)

## Reproduction

1. Build the demo at `default-debug`.
2. Run `tests/engine_demo/test_game_loop` with the disabled regression test enabled
   (`--gtest_also_run_disabled_tests`).
3. Observe failures around frame 1800 (~30 simulated seconds): the sub-step count
   oscillates between 0 and 1 instead of remaining at 1.

## Expected vs. actual

Expected: every `advance(1/60.0)` call returns exactly one substep, indefinitely.
Actual: after ~30 simulated seconds, returns alternate between 0 and 1 due to float
representation drift in the accumulator.

## Root cause hypothesis

`m_accumulator_seconds` is `float`; constitutional article 5 mandates `double` for
simulation accumulators. Each `advance(double)` call narrows through `static_cast<float>`,
losing ~ulp(float) of precision per step.

## Suggested fix (Session 03 stage moment)

Change the storage type to `double`; remove the casts. Add a regression test that runs
3600 frames at 1/60s and asserts every return is exactly 1.
