# BUG-005 — RNG seed silently downcast (CWE-197)

- **Severity:** **high** (security-relevant; entropy regression)
- **Subsystem:** sim
- **File:** `src/engine_demo/sim/rng.cpp`
- **Class:** numeric truncation / CWE-197
- **Anchor for:** Session 04 (Proactive — security track)

## Reproduction

Two `engine_demo::sim::rng` instances seeded with `0x00000000CAFEBABE` and
`0xDEADBEEFCAFEBABE` produce identical streams; they should differ.

## Root cause

`rng::rng(uint64_t)` forwards the seed via `static_cast<uint32_t>`, dropping the upper
32 bits. Any system that derives per-session entropy from a wider source loses 32 bits of
state, weakening determinism guarantees and increasing collision probability across
sessions.

## Fix

Replace the naked downcast with `derive_subseed(seed)` (XOR-fold high half into low half
before passing to the engine). Add a regression test that asserts distinct high words
yield distinct streams.

## Triage note (false-positive companion)

`derive_subseed` itself looks suspicious to a fast Copilot scan — it has a
`static_cast<uint32_t>` on a shifted value. This is **FP-001** in the Session 04 triage
exercise: it widens via XOR, it does not truncate. Learners must dismiss it with a
one-line comment.
