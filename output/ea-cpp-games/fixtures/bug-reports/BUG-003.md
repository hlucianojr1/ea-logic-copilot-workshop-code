# BUG-003 — ECS handle reuse without generation bump (UAF / CWE-416)

- **Severity:** **high** (security-relevant; memory-safety)
- **Subsystem:** ecs
- **File:** `src/engine_demo/ecs/world.cpp`
- **Class:** use-after-free / CWE-416
- **Anchor for:** Session 04 (Proactive — security track)

## Reproduction

1. Build the demo at `default-debug`.
2. Enable the disabled test
   `world.DISABLED_stale_handle_does_not_revive_after_recycle`.
3. The test fails: `is_alive(h_old)` returns true after the slot is recycled.

## Root cause

`world::destroy_entity` flips `slot.alive` to `false` but does **not** increment
`slot.generation`. A subsequent `create_entity` that picks the same slot reuses the prior
generation, allowing a stale handle to dereference logical state belonging to a different
entity.

## Security impact

- Cross-entity component aliasing.
- In a multiplayer scenario, possible authority confusion (one player's stale handle
  manipulating another player's freshly-created entity).
- Potential for crafted inputs to cause memory corruption when component storage is
  pointer-aliased.

## Fix

Add `++m_slots[h.index].generation;` immediately after the `s.alive = false;` line. Add a
regression test asserting `is_alive(h_old) == false` after recycle, and asserting
`h_old.generation != h_new.generation`.
