# BUG-006 — Frame-budget rolling window double-counts on warm-up

- **Severity:** low (cosmetic; affects telemetry only)
- **Subsystem:** frame budget
- **File:** `src/engine_demo/frame_budget.cpp`
- **Class:** off-by-one
- **Anchor for:** Session 03 (Logic Bugs — secondary anchor)

## Reproduction

1. Construct a `frame_budget` with `window_size = 16`.
2. Call `record_sample(10.0)` once.
3. Read `rolling_average()`. Expected: `10.0`. Actual: depends on uninitialized backing
   storage (currently zero-initialized, so accidentally correct on first frame; the bug
   surfaces after the window wraps).

## Root cause

`record_sample` writes at `m_index` and post-increments. `rolling_average` iterates `[0,
m_count)` — but on warm-up the slots beyond `m_index` are stale (or zero) and get folded
into the average.

## Fix

Either iterate from `(m_index - m_count) mod m_window_size` for `m_count` entries, OR
keep a running sum and update on overwrite. Add a regression test that exercises wrap.
