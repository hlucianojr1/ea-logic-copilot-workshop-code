# BUG-010 — Signed/unsigned comparison defeats bounds check

| Field | Value |
|-------|-------|
| **Severity** | High (out-of-bounds read; potential crash or data corruption) |
| **Subsystem** | `sim` |
| **File** | `src/engine_demo/sim/interpolator.cpp` |
| **Class** | `keyframe_interpolator` |
| **Anchor for** | Implicit signed→unsigned conversion makes -1 become SIZE_MAX |

## Reproduction

1. Create a `keyframe_interpolator` and add keyframes.
2. Call `current_value()` WITHOUT calling `start()` first.
3. `m_current_index` is -1 (sentinel). Cast to `size_t` → SIZE_MAX.
4. `m_keyframes[SIZE_MAX % 64].value` reads an arbitrary array slot.
5. If all 64 slots are populated with non-zero data, the "unstarted" read returns garbage.

## Expected vs Actual

| | Expected | Actual |
|--|----------|--------|
| current_value() before start(), empty array | Returns 0.0 sentinel | Returns 0.0 (coincidence — slots are zero-init) |
| current_value() before start(), full array | Returns 0.0 sentinel | Returns m_keyframes[SIZE_MAX % 64].value (non-zero!) |

## Root-cause hypothesis

`m_current_index` is `std::int32_t`, initialized to -1 as a "not started" sentinel.
`current_value()` casts it to `std::size_t`:
```cpp
const auto idx = static_cast<std::size_t>(m_current_index);  // -1 → SIZE_MAX
```

The subsequent bounds check `if (idx < m_count)` becomes `if (SIZE_MAX < m_count)` which
is always false for any realistic `m_count`. The code falls through to
`m_keyframes[idx % kMaxKeyframes].value` — an unintended array slot.

Per [conv.integral]: "If the destination type is unsigned, the resulting value is the least
unsigned integer congruent to the source integer modulo 2^n." So -1 → 2^64 - 1 on LP64.

The compiler warns about this with `-Wsign-compare` (comparing signed and unsigned), but
it's commonly suppressed or ignored.

## Suggested fix

Option A (recommended): Check for the sentinel before casting:
```cpp
double keyframe_interpolator::current_value() const noexcept {
    if (m_current_index < 0 || static_cast<std::size_t>(m_current_index) >= m_count) {
        return 0.0;  // Not started or invalid.
    }
    return m_keyframes[static_cast<std::size_t>(m_current_index)].value;
}
```

Option B: Use `std::optional<std::size_t>` for the index (no sentinel needed).

Option C: Use a separate `bool m_started{false}` flag.

## CWE Reference

- CWE-195: Signed to Unsigned Conversion Error
- CWE-125: Out-of-bounds Read

## Teaching value

Signed/unsigned mismatch is the most common class of bugs flagged by `-Wall` in game code,
and the most commonly suppressed warning. Learners discover:
1. `-Wsign-compare` exists for a reason — don't suppress it.
2. Sentinel values in signed types are dangerous when compared with unsigned.
3. `static_cast<unsigned>(-1)` is SIZE_MAX, not "invalid" — the cast is well-defined but surprising.
4. Copilot identifies sign-conversion issues and suggests type-safe alternatives.
5. Defensive coding: validate sentinel BEFORE type promotion.
