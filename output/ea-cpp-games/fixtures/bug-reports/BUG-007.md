# BUG-007 — Signed-overflow guard elided at -O2

| Field | Value |
|-------|-------|
| **Severity** | High (silent data corruption in Release builds) |
| **Subsystem** | `sim` |
| **File** | `src/engine_demo/sim/timer.cpp` |
| **Class** | `elapsed_timer` |
| **Anchor for** | Signed integer overflow is UB; compiler exploits UB to elide "safety" check |

## Reproduction

1. Build at `-O2` or higher (Release preset).
2. Accumulate > 2,147,483,647 ms via repeated `tick()` calls (simulates 1000× fast-forward for ~35 min).
3. Observe `elapsed_ms()` wraps to a large negative value.
4. At `-O0` (Debug), the overflow guard works by coincidence — the defect only manifests in Release.

## Expected vs Actual

| | Expected | Actual |
|--|----------|--------|
| At -O0 | Guard detects overflow, counter saturates | Counter saturates (works by accident) |
| At -O2 | Guard detects overflow, counter saturates | Guard is elided — counter wraps to INT32_MIN region |

## Root-cause hypothesis

The guard condition `if (m_elapsed_ms + delta_ms > m_elapsed_ms)` relies on signed integer overflow
producing a "wrap" that makes the result smaller. But signed overflow is undefined behavior per
ISO C++ [expr.pre]/4. The optimizer legally reasons: "signed overflow cannot happen →
`(x + positive) > x` is always true → branch is unconditional → remove the `if`."

This is a well-known compiler optimization (documented in GCC `-fstrict-overflow` and Clang's
`-fsanitize=signed-integer-overflow`). The "safety net" becomes a no-op in optimized builds.

## Suggested fix

Option A (recommended): Change `m_elapsed_ms` to `std::uint64_t` and use unsigned arithmetic
(wrapping is defined for unsigned). Remove the fragile guard entirely since overflow takes
~584 million years at 1ms resolution.

Option B: Use `__builtin_add_overflow()` / `std::numeric_limits<int32_t>::max()` to perform the
check without invoking UB:
```cpp
if (delta_ms > 0 && m_elapsed_ms > INT32_MAX - delta_ms) {
    m_elapsed_ms = INT32_MAX;  // saturate
} else {
    m_elapsed_ms += delta_ms;
}
```

## CWE Reference

- CWE-190: Integer Overflow or Wraparound
- CWE-733: Compiler Optimization Removal or Modification of Security-critical Code

## Teaching value

Senior C++ engineers often write "defensive" overflow checks that look correct but invoke
UB — and are silently deleted by the optimizer. This bug teaches:
1. Signed overflow is UB, not wrap-around.
2. "It works in Debug" is not validation.
3. UBSan (`-fsanitize=undefined`) catches this immediately.
4. Copilot can identify UB patterns and suggest well-defined alternatives.
